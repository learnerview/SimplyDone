How Rate Limiting Works
  • Limits are enforced per User ID using a fixed 60-second sliding window backed by Redis.
  • Each job submission increments the counter for the active minute window atomically via Redis INCR.
  • When the window expires the counter resets automatically (Redis TTL).
  • If Redis is unavailable, requests are allowed through (fail-open) with a warning logged.
  • The global default is 60 requests per minute. Configure via simplydone.scheduler.rate-limit.requests-per-minute.
  • Exceeding the limit returns HTTP 429 with a Retry-After header.
